description: >
  <p>Returns <code>true</code> if the key itâ€™s checking is pressed and
  <code>false</code> if not.</p>

  <p><code>keyIsDown()</code> is helpful when checking for multiple different
  key presses.

  For example, <code>keyIsDown()</code> can be used to check if both
  <code>LEFT_ARROW</code> and

  <code>UP_ARROW</code> are pressed:</p>

  <pre><code class="language-js">if (keyIsDown(LEFT_ARROW) &&
  keyIsDown(UP_ARROW)) {
    // Move diagonally.
  }</code></pre><p><code>keyIsDown()</code> can check for key presses using
  strings based on

  <a href="https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key/"
  target="_blank">KeyboardEvent.key</a>

  or <a
  href="https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/code/"
  target="_blank">KeyboardEvent.code</a> values,

  such as <code>keyIsDown('x')</code> or
  <code>keyIsDown('ArrowLeft')</code>.</p>

  <p>Note: In p5.js 2.0 and newer, numeric keycodes (such as 88 for 'X') are no
  longer supported.

  This is a breaking change from previous versions.</p>

  <p>You can still use the p5 constants like <code>LEFT_ARROW</code> which now
  map to string values

  internally rather than numeric codes.</p>
